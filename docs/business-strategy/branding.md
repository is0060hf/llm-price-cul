## 1. 決定事項

| 項目 | 内容 |
|------|------|
| **ブランド名** | **Prestimo**（プレスティモ） |
| **ドメイン** | **prestimo.io** |
| **由来** | Presto（即座に）+ Estimo（Estimate の短縮）= 「即座に見積もる」 |
| **音節数** | 3 音節（プレス・ティ・モ） |

### 選定理由

- **拡張性**: LLM/AI に縛られない造語のため、LLM コスト見積もり → システム開発全体の見積もりへの拡張に対応可能
- **語感**: イタリア語的な響きで洗練されている。3 音節で覚えやすい
- **即時性の表現**: Presto（即座に）が込められており、リアルタイム見積もりという最大の差別化要素と合致
- **独自性**: 造語のため、検索で他サービスと衝突しにくい
- **口頭伝達**: 「プレスティモで見積もり出して」と自然に使える

### 補足

- 「Prestimo」はポルトガル語・スペイン語圏で「貸付・融資」を意味する単語として存在する
- 主要市場が日本であるため実害はほぼなく、金融・コスト系のニュアンスとして方向性に矛盾しない

## 2. ブランド名に求められた要件

将来像（AI 搭載の見積もりプラットフォーム）を踏まえ、以下の要件で選定した。

| 優先度 | 要素 | 理由 |
|--------|------|------|
| **必須** | 「見積もり」のニュアンス | プロダクトの本質。Phase 1 でも最終形でも変わらない |
| **必須** | スケールしても陳腐化しない広さ | LLM コスト → システム開発全体に拡張予定 |
| **推奨** | BtoB SaaS としての信頼感 | 企業の内部原価を預かるサービスになる |
| **推奨** | 「速さ・即時性」のニュアンス | リアルタイム見積もりが最大の差別化要素 |
| **任意** | 日本語話者に親しみやすい | 主要市場が日本 |

## 3. ドメイン選定

### 決定: prestimo.io

| 観点 | 評価 |
|------|------|
| TLD の印象 | テック系スタートアップの定番。BtoB SaaS として適切 |
| 口頭伝達 | 「プレスティモ・ドット・アイオー」で自然に伝わる。ハイフンなし |
| 記憶性 | ブランド名 + `.io` でシンプル |

### 検討した他の候補

| # | ドメイン | 不採用の理由 |
|---|---------|-------------|
| 1 | `pres-timo.com` | ハイフン入りは口頭伝達時に「prestimo.com」と誤入力されるリスク |
| 2 | `prestimo.app` | ツール感は出るが、SaaS プラットフォームとしてはやや軽い印象 |
| 3 | `prestimo.dev` | 開発者向け TLD。営業・PM がターゲットには不向き |
| 4 | `prestimo.jp` | 日本特化すぎる。将来的なグローバル展開の可能性を残す |
| 5 | `getprestimo.com` | `.com` にこだわるなら有力だが、`prestimo.io` の方がシンプル |

> **注意**: ドメインの空き状況は 2026-02-15 時点で未確認。取得前に必ず確認すること。

## 4. 検討経緯（アーカイブ）

### 4.1 初回提案（Phase 1 前提・不採用）

将来像が共有される前に、LLM コスト見積もりに特化した名前として提案したもの。
将来像の共有により「LLM/AI に縛られすぎる」と判断し不採用。

| # | ブランド名 | 不採用理由 |
|---|-----------|-----------|
| A1 | LLMCostCap | 「LLM」に縛られる。Phase 2 以降と合わない |
| A2 | AgentCostLab | 「Agent」が AI エージェントに限定される |
| A3 | LLM見積もりツール | ブランドとしての記憶性が弱い |
| B1 | Capra | 単体では意味が伝わらない |
| B2 | Costimize | 「Customize」と誤読リスク |
| B3 | Tokenary | LLM に縛られる |
| C1 | ミツモLLM | 「LLM」に縛られる |
| C2 | AI費用キャップ | ブランドとしての洗練さに欠ける |
| C3 | LLMキャルク | 「LLM」に縛られる |

### 4.2 再提案（将来像を踏まえた候補）

| カテゴリ | 候補 | 採否 |
|---------|------|------|
| 見積もり × 速さ | Qstimate、Estimo、FlashEstimate | Estimo の要素を Prestimo に採用 |
| 見積もりプラットフォーム | Mitsumori、EstiPro、QuoteCraft | 不採用 |
| 造語型 | Cevio、Prestly、Velocost | Prestly の要素を Prestimo に採用 |

最終的にプロダクトオーナーが Estimo + Prestly を組み合わせた **Prestimo** を提案し、採用された。
