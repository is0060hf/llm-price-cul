## 2. 議論の経緯

### 2.1 システム理解（コードベース・要件書の精査）

要件定義書（`docs/requirement.md`）およびコードベースの実装内容を精査し、以下を確認した。

- **目的**: AI エージェントを Web システムに組み込む際の月額ランニングコストの上限値を算定するツール
- **2 つの算定モード**: ざっくり算定（5 項目入力）/ 詳細算定（36 項目入力）
- **対応プロバイダー**: OpenAI / Anthropic / Google
- **主要機能**: コスト算出、比較表、成長シナリオ予測、システムプロンプト推定ヘルパー
- **技術スタック**: Next.js (App Router) + Neon (PostgreSQL) + Drizzle ORM + shadcn/ui

### 2.2 ターゲット層・差別化の分析

要件書のセクション 1.3 に基づき、ターゲット層を 2 層に整理した。

| ターゲット | 人物像 | 対応モード |
|-----------|--------|-----------|
| 第 1 ターゲット | SIer / 受託開発企業の営業・PM・プリセールス | ざっくり算定 |
| 第 2 ターゲット | 事業会社のエンジニア・テックリード | 詳細算定 |

競合（LLM 料金比較サイト、スプレッドシート自作）との差別化ポイントとして以下を特定した。

1. 「単価表」ではなく「月額見積」を出すこと
2. マルチエージェント構成全体のコストを積み上げ計算すること
3. 保守的な上限値算定により「これ以上にはならない」という安心感を提供すること

### 2.3 プロダクトの将来像（プロダクトオーナーより共有）

プロダクトオーナーより、以下の将来像が共有された。

#### 現在（Phase 1）
- LLM ランニングコストの上限見積もりツール（無料・認証なし）

#### 最終形
- **SIer / 受託開発企業の営業・PM 向けの、AI 搭載見積もりプラットフォーム（BtoB SaaS）**

具体的には:

| 要素 | 内容 |
|------|------|
| 対象範囲 | システム開発費全体（LLM/AI + インフラ + 開発工数 + 保守運用） |
| 強み | AI 系コスト見積もりの精度が特に高い |
| AI 搭載 | LLM を組み込み、商談中にリアルタイムで概算見積もりを生成 |
| マルチテナント | ログインユーザー（企業）ごとに利益率・バッファ・人月単価を管理 |
| 秘匿性 | 内部原価とクライアント提示価格を分離。社内用画面と顧客向け画面が別 |
| 利用場面 | 商談の場で PM / 営業がリアルタイムに見積もりを提示 |

#### 将来像を踏まえた解決すべきペインポイント

> 見積もり作業自体はエンジニアを巻き込んで結構時間がかかる作業である。このペインポイントを AI で解消し、商談中にリアルタイムで概算見積もりを作成できるようにする。

### 2.4 ブランド名の検討

#### 初回提案（Phase 1 前提）

将来像が共有される前に、LLM コスト見積もりに特化した名前として以下を提案した。

- LLMCostCap、AgentCostLab、ミツモLLM 等

#### 将来像を踏まえた再評価

将来像の共有により、上記候補は全て「LLM/AI に縛られすぎる」と判断し、再提案を行った。

ブランド名に求められる要件を再定義:

| 優先度 | 要素 | 理由 |
|--------|------|------|
| 必須 | 「見積もり」のニュアンス | プロダクトの本質 |
| 必須 | スケールしても陳腐化しない広さ | LLM → システム開発全体に拡張 |
| 推奨 | BtoB SaaS としての信頼感 | 企業の内部原価を預かるサービス |
| 推奨 | 「速さ・即時性」のニュアンス | リアルタイム見積もりが最大の差別化 |

#### 再提案候補

3 カテゴリ 9 案を提案:

- **「見積もり × 速さ」型**: Qstimate、Estimo、FlashEstimate
- **「見積もりプラットフォーム」型**: Mitsumori、EstiPro、QuoteCraft
- **造語型**: Cevio、Prestly、Velocost

#### プロダクトオーナーからの提案

プロダクトオーナーより、Estimo（Estimate 短縮）と Prestly（Presto = 即座に）を組み合わせた **「Prestimo」** が提案された。

### 2.5 ドメイン名の検討

- プロダクトオーナーから `pres-timo.com` が提案された
- ハイフン入りドメインは口頭伝達時の課題（「プレスティモドットコム」と聞いた相手が `prestimo.com` と打つ可能性）が指摘された
- 代替候補として `prestimo.io`、`prestimo.app`、`prestimo.jp`、`getprestimo.com` 等を提示

## 3. 決定事項

| # | 決定事項 | 内容 |
|---|---------|------|
| 1 | **ブランド名** | **Prestimo**（プレスティモ） |
| 2 | **ドメイン** | **prestimo.io** |
| 3 | **名前の由来** | Presto（即座に）+ Estimo（Estimate の短縮）= 「即座に見積もる」 |
| 4 | **コンセプト** | 3 音節で覚えやすく、造語のため独自性が高い。LLM/AI に縛られず将来の拡張にも対応 |

### 補足事項

- 「Prestimo」はポルトガル語・スペイン語圏で「貸付・融資」を意味する単語として存在する。主要市場が日本であるため実害はほぼなく、金融・コスト系のニュアンスとして方向性に矛盾しない
- ドメインの空き状況は未確認。取得前に要確認
